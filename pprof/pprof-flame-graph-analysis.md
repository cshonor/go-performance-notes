# pprof 火焰图详解与实战

## 为什么火焰图必须和 pprof 绑定学习

- 火焰图是 pprof 的**核心可视化手段**：把“耗时/调用栈”变成一眼能读的结构
- 面试常考：“火焰图怎么看”“宽度/高度分别代表什么”“怎么从图里找到优化点”

## 火焰图怎么看（最小必要认知）

- **每个块**：一个函数/栈帧（取决于视图）
- **宽度**：消耗（CPU 时间或采样权重），越宽越贵
- **上下层级**：调用关系（父调用在下，子调用在上）
- **关键任务**：找到“最宽的那条调用链”并回到代码验证

## 推荐使用方式（pprof 原生）

> 目标：不用第三方工具也能稳定生成与分享。

- 用 `go tool pprof -http=:0 <binary> <profile>` 启动 UI
- 在 UI 里选择 flamegraph 相关视图，配合 `top/list` 做证据闭环

## 实战场景（你要能在图上识别这些信号）

- **CPU 耗时**：业务计算重、序列化/反序列化、正则、加密、JSON 等
- **阻塞/锁竞争**：大量时间消耗在同步与等待（需结合 mutex/block）
- **GC 相关**：分配过多导致 GC 开销（需结合 alloc/heap + 逃逸分析）
- **调用链异常**：某个“看似无关”的中间层变宽（往往是抽象/接口/反射导致）

## 面试输出模板（建议照着背）

- 我会先用 CPU profile + 火焰图找最宽调用链
- 再用 `top/list` 定位到具体函数与代码行
- 如果怀疑 GC/分配：补采 alloc/heap，并用逃逸分析解释根因
- 如果怀疑竞争/阻塞：补采 mutex/block/goroutine 证明等待点
- 最后做前后对比，用数据证明优化收益

# sync.Pool 面试考点总结

## 1) sync.Pool 是什么，解决什么问题

- 复用临时对象，减少分配与 GC 压力
- 常见于高频请求路径的 buffer/临时对象复用

## 2) 为什么 Pool 不能当“对象/连接池”（高频）

- Pool 不保证对象一直存在（可能被 GC 清空）
- Pool 适合“可丢弃可重建”的对象，不适合强生命周期资源

## 3) 为什么要和 P 绑定

- 提高并发性能、减少争用、提升局部性

## 4) 适用/不适用场景

- 适用：短生命周期、高频创建、可 reset 的对象
- 不适用：大对象、长生命周期对象、外部资源句柄、需要强保证的缓存

## 5) 如何验证收益（必须会讲）

- pprof 前后对比：alloc/heap/CPU
- benchmark 指标：`allocs/op`、`B/op`、吞吐
- 线上关注：GC 频率、P99 延迟

# sync.Pool

## 核心作用

复用临时对象，减少分配次数与 GC 压力，常用于高频、短生命周期对象（buffer、临时结构体等）。

## 关键特性（容易踩坑）

- `sync.Pool` **不是对象池语义保证**：对象可能在 GC 后被清空；不适合放“必须持久存在”的对象
- 适合“可丢弃、可重建、短生命周期”的临时对象
- 复用收益通常体现在：减少 `allocs/op`、降低 `B/op`、降低 GC 频率与尾延迟

## 面试考点

- Pool 的大致实现与行为（为何 GC 会清空、为什么不能当连接池）
- 与 GMP/本地缓存的关系（为何并发下仍然高效）
- 典型优化场景：`bytes.Buffer` / `[]byte` 缓冲复用、编解码临时对象复用

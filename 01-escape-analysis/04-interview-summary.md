# 逃逸分析面试考点总结

## 1) 逃逸分析是什么，有什么用

- 判断变量分配在栈还是堆
- 指导减少堆分配与 GC 压力，改善吞吐与尾延迟

## 2) 什么情况下变量会逃逸（高频）

- 返回局部变量指针/引用
- interface 装箱（跨接口边界）
- 闭包捕获变量
- 反射、`fmt`、动态调用等让编译器难证明不逃逸的路径
- 切片/Map 扩容等导致分配（结合具体代码）

## 3) 如何减少逃逸（优化方法）

- 减少不必要的接口边界与装箱
- 避免不必要闭包捕获
- 减少反射、`fmt` 这类高开销/保守路径
- 让编译器更容易分析（减少过度抽象，必要时写直白代码）

## 4) 逃逸与 GC 的关系（怎么讲清楚）

- 逃逸越多 → 堆对象越多 → GC 需要扫描/回收的对象越多
- 这会导致更高的 GC 频率/开销，可能造成延迟抖动
- 结论必须用 pprof（alloc/heap/CPU）做证据验证

## 5) 常见追问

- `-gcflags="-m -m"` 输出怎么看？你关注哪些提示？
- “不逃逸就一定更快吗？”（不一定，要看瓶颈与可读性/复杂度权衡）

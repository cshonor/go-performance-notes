## 核心作用

判断变量分配在 **栈** 还是 **堆**，帮助减少堆分配、降低 GC 压力与尾延迟。

## 常用入口

- `go build -gcflags="-m" ./...`
- `go test -gcflags="-m" ./...`

## 常见逃逸触发点（面试高频）

- 返回局部变量指针（生命周期超出当前栈帧）
- 把值装进 `interface{}`（尤其是非指针小对象）导致装箱与逃逸
- 闭包捕获变量（变量地址被延长）
- `[]byte`/`string` 转换、切片扩容等间接导致分配（视具体代码路径）
- 通过反射、`fmt`、`append` 等让编译器难以证明不逃逸

## 优化抓手

- 尽量传值/用结构体聚合，减少 `interface{}` 边界
- 复用临时对象（配合 `sync.Pool`）但要确认收益与生命周期
- 让编译器“看得懂”：减少过度抽象，避免不必要的闭包与反射

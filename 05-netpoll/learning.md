# netpoll 学习路径（第五优先级）

## 为什么第五个学

- netpoll 是 Go 网络 I/O 非阻塞的核心，通常需要结合：
  - trace：可视化网络 I/O 事件、阻塞与唤醒
  - pprof：分析网络服务的热点与瓶颈（CPU/阻塞/竞争/分配）
- 面向目标：网络性能优化与高并发服务（面试也很爱问）

## 学习路径（闭环）

### 1) 底层原理

- I/O 多路复用：基于 epoll/kqueue（不同平台实现不同）
- netpoll 的职责：把 fd 就绪事件转换为 goroutine 可运行与唤醒

### 2) 与 GMP 结合（必须打通）

- 网络 I/O 等待时，G/M/P 的行为变化（谁阻塞、谁继续调度）
- 事件到来后 goroutine 如何被唤醒、何时重新变为 runnable

### 3) 实战优化（pprof + trace 联动）

- 用 pprof 找热点与瓶颈（可能是序列化、日志、锁、分配、系统调用）
- 用 trace 证明“等待发生在哪里”（netpoll / syscall / 竞争 / GC）
- 结合指标形成证据链，再做最小优化并验证

## 面试考点（高频）

- Go 网络 I/O 模型是什么
- netpoll 与 GMP 的关系（为什么网络 I/O 不会阻塞 P）
- epoll 原理与基本概念（fd、事件、就绪、边沿/水平触发等）

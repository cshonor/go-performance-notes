# 逃逸分析实战案例（10 大场景复现 + 优化对比）

## 目标

用“复现逃逸 → 解读输出 → 优化代码 → pprof 验证”的闭环，真正掌握逃逸规则与优化手法。

## Demo 清单（建议逐个做成可运行样例）

> 每个 demo 都输出两份：**逃逸版本** / **不逃逸（或更少逃逸）版本**，并附 `-m -m` 关键输出。

1. 返回局部变量指针导致逃逸
2. interface 装箱导致逃逸（常见于 `fmt`、抽象接口边界）
3. 闭包捕获变量导致逃逸
4. 使用 `append` 导致底层数组分配（结合容量变化）
5. `[]byte` / `string` 转换触发分配（视写法）
6. map/slice 作为返回值/跨 goroutine 使用导致逃逸
7. 反射（`reflect`）路径导致逃逸
8. 大对象传参/返回导致堆分配（结合实际结构大小）
9. 方法接收者/接口调用导致的间接逃逸
10. 逃逸与内联（inlining）的关系（对比有无内联）

## 验证方式（必须和 pprof 绑定）

- 用 `pprof` alloc/heap 验证改动是否减少：
  - `allocs/op`
  - `B/op`
  - GC 次数/时间（间接指标）

## 标准输出模板（每个场景都按这个写）

- 现象：为什么它可能会逃逸
- 命令：`go test/build -gcflags="-m -m"`
- 关键输出：哪一行提示了逃逸，原因是什么
- 优化：怎么改，为什么能减少逃逸
- 验证：pprof 前后对比结论

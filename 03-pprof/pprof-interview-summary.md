# pprof 面试考点总结

## 1) pprof 能解决什么问题

- CPU 热点定位（哪里最耗时）
- 内存问题（内存为什么涨、哪里分配多）
- goroutine 问题（泄漏、死锁、卡住）
- 竞争/阻塞（锁竞争、chan/select 阻塞）

## 2) 6 大 profile 什么时候用

- **CPU**：怀疑“算得慢 / 热点函数”
- **Heap**：怀疑“内存涨 / OOM / 长生命周期对象”
- **Allocations**：怀疑“分配过多导致 GC 压力”
- **Goroutine**：怀疑“协程泄漏 / 卡住 / 死锁”
- **Mutex**：怀疑“锁竞争导致吞吐低/延迟高”
- **Block**：怀疑“等待/阻塞导致请求慢”

## 3) 火焰图怎么看（高频）

- 宽度代表消耗（采样权重），越宽越贵
- 看最宽的调用链，再回到 `list` 看代码行
- 结论必须配合可复现负载与前后对比，不要“凭一张图拍脑袋”

## 4) 线上排查的完整流程（建议背诵版）

1. 明确现象与指标（QPS、P95/P99、CPU、heap、goroutines、GC）
2. 复现稳定负载（或在线低风险采样）
3. 选择 profile 并采集（一次只回答一个问题）
4. 分析（top → list → 调用链/图谱 → 形成假设）
5. 最小改动优化
6. 再采集并对比验证
7. 输出结论与回归点（避免复发）

## 5) 常见追问（你需要能回答）

- CPU profile 与 heap/alloc 的区别是什么
- 为什么 alloc 很高会导致 GC 变差（堆增长、GC 频率、STW/辅助 GC）
- goroutine profile 如何判断泄漏 vs 正常等待
- mutex/block 看到“等待多”后，下一步怎么优化（缩临界区/分片/减少共享/调整并发模型）

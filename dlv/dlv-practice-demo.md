# dlv 实战案例（死锁/源码调试/线上复现）

## 目标

用 dlv 把“猜测”变成“证据”：在关键路径断点，观察 goroutine、锁、chan 的真实状态。

## Demo 1：死锁排查（经典）

- **现象**：程序卡住不动，CPU 很低，goroutine 不再前进
- **步骤建议**：
  - 先用 pprof goroutine dump 找堆栈集中点
  - 再用 dlv 复现，在关键锁/chan 处断点
  - 切换 goroutine 查看等待点与持锁点
- **输出**：死锁环路的证据链（哪两个/多个 goroutine 在互相等待）

## Demo 2：调试 runtime 源码路径（chan 阻塞）

- **目标**：断点跟进一次 send/recv 的阻塞与唤醒路径
- **输出**：关键状态变化（阻塞点、唤醒点、调度恢复点）

## Demo 3：线上问题复现（流程演练）

- **步骤**：收集现场信息 → 本地/预发复现 → dlv 验证 → 修复与回归

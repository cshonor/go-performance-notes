# dlv 底层原理（调试器工作机制概览）

## 你需要掌握的“最小原理集”

- 调试器的核心能力：控制执行（暂停/继续/单步）、读取状态（变量/堆栈/寄存器/线程）
- Go 程序的调试需要符号信息：构建方式与环境一致性很关键
- 调试 runtime 时，本质是观察 goroutine/线程/调度相关状态在关键路径的变化

## 为什么它能解决 pprof/trace 解决不了的问题

- pprof/trace 提供统计/事件证据，但不一定能精确到某个分支条件
- dlv 可以在“你怀疑的点”停住，检查当时的变量与状态，验证根因
